**Full Title:** Security Benchmarking of AI-Generated Web Application Code: An OWASP-Based Comparative Study of Coding LLMs

## Project Overview
This repository contains the research data, evaluation code, and manuscript drafts for a comparative study on the security of code generated by Large Language Models (LLMs). 

### 🎯 Objective
We evaluate **Gemini 3.5 Flash**, **DeepSeek V4 Flash**, **Kimi K2.5**, and **GPT-5.4-mini** against standard software engineering tasks to identify common security vulnerabilities mapped to the **OWASP Top 10**.


## Repository Structure
- `data/`: Raw code generated by each LLM, organized by model and prompt.
- `results/`: Security scan reports (Semgrep, OWASP ZAP) and data analysis.
- `paper/`: Drafts of the research paper sections.
- `prompts.md`: The 10 standardized prompts used for the experiment.
- `brainstorm.md`: Initial brainstorming and project planning.

## How to Run the Experiments
1. **Generation:** Use the prompts in `prompts.md` in the target LLMs.
2. **Storage:** Save outputs in `data/<model>/prompt<n>/`.
3. **Analysis:** Run Semgrep and ZAP as per the protocol in the `brainstorm.md` or the implementation plan.

## Research Questions
1. To what extent do modern LLMs generate code with OWASP Top 10 vulnerabilities?
2. Which LLM produces the most secure code for web applications?
3. Does the complexity of the task correlate with the density of vulnerabilities?
